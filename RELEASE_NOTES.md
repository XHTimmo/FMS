# Studio Financial System v1.2.0 Release Notes

**版本号**: v1.2.0  
**发布日期**: 2026-03-19  

## 🚀 变更摘要
本次版本是一次重大的功能与体验升级，主要聚焦于**数据导出引擎的全面重构**、**会费与账单双向同步机制**以及**代码库质量的大幅提升**。我们彻底移除了旧版基于系统截图的粗糙导出方式，引入了支持真实数据与样式无损还原的高质量多格式（PDF/Excel/PNG）导出。同时，针对核心业务逻辑进行了深度的代码审查，消除了数十处潜在的类型安全隐患，增强了系统的稳定性和可靠性。

## ✨ 详细更新列表

### 1. 核心功能更新 (Features)
- **全新数据导出引擎**：
  - **高清 PDF 与 Excel 导出**：基于 Electron 隐藏窗口渲染与 ExcelJS，实现了看板图表、表格明细数据的无损多格式导出。保证了分页、字体、颜色及边距与原页面完美一致。
  - **自定义 PNG 截取**：引入 `html2canvas` 结合 ECharts 高清图表，实现基于 DOM 的精准局部截图（支持全屏、核心数据、仅表格三种区域选择），并支持自定义分辨率。
  - **全量数据获取**：解除了原先分页查询导致的数据遗漏，现可一键导出指定日期范围内的所有账单与财务明细。
- **进度提示与重试机制**：为大文件导出新增了实时进度条与失败重试确认框，显著提升了用户体验。
- **会费与账单双向流转**：完善了会费记录与系统账单交易的联动逻辑，支持缴费状态自动同步与交易撤销级联回滚。

### 2. 问题修复 (Bug Fixes)
- **修复导出图片截断问题**：解决了旧版本中前端 Flex/Grid 布局在截取 PNG 时右侧出现灰色背景及内容截断的严重 Bug。
- **修复 Excel 图表插入失败**：修正了 `dashboard.ts` 中 Base64 数据头处理逻辑，确保统计图表能够成功嵌入到生成的 Excel 报表中。
- **修复 PDF 加载空白**：重构了 `printToPDF` 相关的临时文件加载机制，移除了不支持的参数，确保生成的 PDF 能够正常预览和保存。
- **修复 HTML 模板转义错误**：清理了报表生成模板中多余的反斜杠（`\`），确保生成的临时 DOM 能够正确挂载图表。

### 3. 代码质量与性能优化 (Optimizations)
- **全面移除 `any` 增强类型安全**：重构了 `src/main/ipc/` 下的所有核心业务处理器（包含 `dashboard.ts`、`transactions.ts`、`members.ts` 等），引入了 `TransactionDetailRow`、`MemberRow` 等强类型接口，实现了 100% 静态类型覆盖。
- **规范化异常处理机制**：将全局 IPC 处理器中的 `catch (error: any)` 改造为更为现代和安全的 `catch (error: unknown)`，并通过类型收窄规避了非标准异常引发的崩溃。
- **构建产物深度清理**：彻底清理了项目中冗余的缓存文件（如 `.idea`, `dist-electron`, 遗留 `node_modules` 等），释放了超 1GB 的磁盘空间。
- **CI/CD 自动化发布增强**：优化了 GitHub Actions 流水线，现在通过打 `v*` 标签并推送后，在测试和编译通过时将自动通过 `electron-builder` 发布**正式的 GitHub Release**，无需手动干预。
